# Changelog

Все заметные изменения проекта. Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/),
версионирование — [SemVer](https://semver.org/lang/ru/).

## [0.4.3-beta] — 2026-06-15

### Добавлено
- **Прозрачный терминал обновления.** Диалог обновления теперь показывает весь
  ход: подключение, версию до/после, вывод git и pip, а при сбое — полный
  traceback с сервера. Обработка `onclose`/`onerror` на фронте: если соединение
  упало до завершения, выводится понятное сообщение вместо немой тишины.

[0.4.3-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.4.3-beta

## [0.4.2-beta] — 2026-06-15

### Исправлено
- **`No module named 'PIL'` при работе с QR-кодами.** Пакет `qrcode` рендерит PNG
  через Pillow, но Pillow не был в зависимостях — на чистом контейнере QR падал
  (например, при создании ключа Outline). Добавлен `pillow` в `requirements.txt`.
  Плюс генерация QR сделана необязательной: если Pillow нет, операция не падает,
  просто остаётся без картинки (ссылка всё равно доступна).

[0.4.2-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.4.2-beta

## [0.4.1-beta] — 2026-06-15

### Исправлено
- **Обновление молча не применялось.** `git pull --rebase` падал с кодом 128,
  если рабочее дерево на сервере «грязное» (изменённые права файлов, недокачанный
  rebase от прошлых попыток). Теперь обновление идёт через `git fetch` +
  `git reset --hard origin/main` — устойчиво к любому состоянию дерева. Добавлен
  `safe.directory` (от ошибки «dubious ownership»). То же в CLI `vpnpilot-update`.
  Причина воспроизведена локально.

[0.4.1-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.4.1-beta

## [0.4.0-beta] — 2026-06-15

### Добавлено
- **Управление ключами Outline прямо в панели.** Менеджер серверов теперь
  показывает список ключей доступа (`ss://` ссылки для подключения клиентов) с
  QR-кодами, создание ключа с именем и удаление — всё через Outline API.

### Изменено
- **Outline: больше не путаница со ссылкой.** Раньше показывался только
  управляющий ключ `{"apiUrl":...,"certSha256":...}` — его принимали за «ссылку
  доступа», хотя он для Outline Manager, а не для подключения. Теперь на первом
  месте `ss://` ключи для клиентов, а управляющий ключ вынесен в сворачиваемый
  блок «Для администратора» с пояснением. Проверено вживую (создание/список/
  удаление ключей через API).

[0.4.0-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.4.0-beta

## [0.3.1-beta] — 2026-06-15

### Исправлено
- **Кнопка «Обновить» зависала.** `systemctl restart` убивал процесс uvicorn до
  отправки финального сообщения — фронт не получал сигнал и не перезагружался.
  Теперь сначала шлём `done`, потом перезапускаем сервис отвязанно через
  `systemd-run` (переживает наше завершение). Страница обновляется сама.
- **Контейнер Proxmox брал имя хоста вместо `vpnpilot`.** Переменная `HOSTNAME`
  в скрипте конфликтовала со стандартной bash-переменной (= имя Proxmox-хоста).
  Переименована в `CT_HOSTNAME`.

[0.3.1-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.3.1-beta

## [0.3.0-beta] — 2026-06-15

### Добавлено
- **Кнопка «⏹ Остановить»** во время установки — прерывает зависшую/идущую
  установку за пару секунд (рвёт SSH-канал, не ждёт таймаута). Эндпоинт
  `POST /api/install/{session_id}/cancel`. Проверено вживую.
- **Сохранённые серверы** — IP/порт/пользователь/пароль запоминаются после
  успешной проверки соединения или запуска установки. В форме появляется список
  «💾 Сохранённые серверы»: клик подставляет данные, не нужно вводить заново.
  Эндпоинты `GET /api/hosts`, `DELETE /api/hosts/{host}/{port}`.

[0.3.0-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.3.0-beta

## [0.2.1-beta] — 2026-06-15

### Исправлено
- **Outline ставится с первого раза.** Корневая причина вечного `Starting
  Watchtower ... FAILED`: контейнер `watchtower` от прошлой попытки занимал имя,
  `install_server.sh` падал на этом шаге **до записи `apiUrl`** — ключ для Outline
  Manager получался неполным. Теперь перед установкой старые контейнеры
  (`shadowbox`, `watchtower`) сносятся, и скрипт проходит штатно.
- `outline_config` собирает ключ **напрямую из контейнера** (порт + префикс из
  env, certSha256 из сертификата), а не из неполного `access.txt` — рабочий ключ
  получается, даже если скрипт где-то споткнулся. Проверено вживую.

[0.2.1-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.2.1-beta

## [0.1.0-beta] — 2026-06-15

Первый публичный бета-релиз.

### Добавлено
- Веб-панель установки VPN по SSH (IP + пароль), авто-определение ОС.
- 7 протоколов: WireGuard, OpenVPN, VLESS+Reality, 3X-UI, Shadowsocks, Outline, IKEv2.
- QR-коды и конфиг-файлы сразу после установки.
- Управление клиентами (добавить/удалить) для поддерживаемых протоколов.
- Полный снос VPN с сервера.
- **Переустановка начисто** для сохранённых серверов (чинит зависшие установки).
- **Проверка обновлений** из UI: сверка с GitHub + `git pull` + перезапуск.
- CLI-команда `vpnpilot-update` как фолбэк.
- Мульти-провайдерный AI-помощник (Groq / Gemini / OpenRouter / DeepSeek).
- Установщики: `install.sh` (контейнер/VM) и `proxmox-install.sh` (создаёт LXC).

### Исправлено
- Зависание SSH: `run_command_stream` больше не блокируется навсегда после
  таймаута; добавлен idle-таймаут (прерывает молчащий установщик, как Outline).
- Outline: игнорируется ошибка Watchtower в unprivileged LXC, проверяется
  только Shadowbox.
- `/api/version` устойчив к приватному репозиторию (локальная версия не зависит
  от сетевой проверки); `GIT_TERMINAL_PROMPT=0` — git не виснет на запросе пароля.

[0.1.0-beta]: https://github.com/hagsy2/vpnpilot/releases/tag/v0.1.0-beta
